idea: better slack + github notification.

topic: slack

pain points with current integration:

  • when you have github notifications going to a slack channel it quickly becomes too noisy for any other conversation
  • if there is a conversation going on in a github issue, and then people see the notification in slack and start discussing it there, we have a fragmented discussion

notes:

  • what if we had a dedicated github notiication channel that was understood to be read-only?

  • or what if, instead of spewing out every commit and comment, it gave some other kind of helpful visual indication of:

    • progress (commits, comments)
    • spikes (lots of commits / comments in a short time)
    • use ascii-art to show basic graphs showing spikes?

prototype:

  • every 5 minutes, bot gives a status update

  • only if something has happened during that time

  • commits by dev

  • new issues / PRs

  • comments by issue

  • useful stuff:

    • https://github.com/rvagg/github-webhook-handler for webhooks
    • ngrok for developing github webhooks locally
    • https://github.com/jstrace/chart for charts
