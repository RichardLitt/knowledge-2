Q. what options are there for doing a local record-and-playback of user interactions in a browser? Something not as heavyweight as selenium. If there was a low-effort way to record+playback it could be a useful dev tool, something less automated but more long lasting than selenium tests.

  • if it's always recording, it's easy to go back and find the bit you want to replay

prototype 1

  • electron-based browser or maybe chrome extension if that's easier

  • everything you do in the browser gets logged

    • mouse clicks
    • keyboard
    • page change
    • timing
  • logs go to a human-readable file which can be edited and replayed
  • so you can be working on a site and fix a bug or complete an interaction, then find that place in the log and create a replay of the part you want
  • 1-click screencap (and show diff next time)
  • text / visual annotation so you can create walk-throughs for other people
